# Hamnix Bitmap Font Store (Phase 4d)

This directory vendors three small public-domain bitmap fonts in BDF
2.1 format, used by the hamUI/DE text-draw path (`lib/font_bdf.ad` +
`user/hamUId.ad`'s `raster_text`). All three are generated by
`scripts/gen_bdf_fonts.py` from data that is either non-copyrightable
or algorithmically synthesised in-tree.

## Fonts

| File                       | Family | Size  | License        |
|---------------------------|--------|-------|----------------|
| `hamnix-mono-8x16.bdf`    | mono   | 8x16  | Public Domain  |
| `hamnix-sans-6x10.bdf`    | sans   | 6x10  | Public Domain  |
| `hamnix-serif-6x12.bdf`   | serif  | 6x12  | Public Domain  |

The mono font is a re-encoding into BDF of the IBM CGA/VGA ROM 8x16
glyph data already embedded in `user/hamUId.ad`. ROM-typeface bitmaps
are uncopyrightable under US Federal Code; the recoding into BDF
inherits that public-domain status.

The sans and serif fonts are generated from a small in-tree
algorithmic 5x7 stencil table inside `scripts/gen_bdf_fonts.py` (no
third-party data). The script applies a 1-pixel top/bottom serif
spread to derive `serif` from the same stencil.

## Format subset

`lib/font_bdf.ad` parses the minimal BDF 2.1 subset emitted by the
generator:

```
STARTFONT 2.1
FONTBOUNDINGBOX W H xoff yoff
CHARS N
STARTCHAR <name>
ENCODING <codepoint>
DWIDTH <w> 0
BBX <w> <h> <xoff> <yoff>
BITMAP
<HH>*<h>
ENDCHAR
...
ENDFONT
```

PROPERTIES blocks, SWIDTH-only glyphs, vertical metrics, and writing
direction are tolerated (skipped) but not honoured. Codepoints above
0xFF are tolerated but never matched at render time (hamUI is ASCII-
only in v1).

## Regenerating

```
python3 scripts/gen_bdf_fonts.py
```

The generator is idempotent; commit the resulting `.bdf` files.
