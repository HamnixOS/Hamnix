# Native Browser (hambrowse) ‚Äî W3C / ECMAScript Conformance Roadmap

From a 14-agent parallel audit (2026-07-16). Dual-target, host-iterated (~40 gates). ~30-50% conformant/area; full conformance needs real spec ALGORITHMS per subsystem ‚Äî enabled by the modular lib/web/ tree refactor.

## ECMAScript rounds landed (lib/web/js/*, host gates in scripts/test_jsengine_*_host.sh)

- R1-6: templates/tagged templates, classes + fields + accessors, Symbol + well-known symbols, change-array-by-copy, regex named groups, generators (`function*`/`yield`/`yield*`, bounded-eager model) + the iteration protocol, Map/Set, optional chaining/nullish, for-of, Object/Reflect-ish statics.
- **R7** (this round):
  - **Generator methods** ‚Äî `*name(){}` in object literals and class bodies (incl. `static *m(){}`). `test_jsengine_generators_host.sh`.
  - **Reflect** ‚Äî get/set/has/ownKeys/getPrototypeOf/setPrototypeOf/defineProperty/deleteProperty/apply/construct (thin wrappers over core ops). Also fixed the `in` operator to walk the prototype chain. `test_jsengine_reflect_host.sh`.
  - **Proxy** ‚Äî `new Proxy(target, handler)` with the get/set/has/deleteProperty/ownKeys traps routed through member-get/set, `in`, `delete`, Object.keys/Reflect.ownKeys; absent trap forwards to target. Deferred: apply/construct/defineProperty/getOwnPropertyDescriptor/get(set)PrototypeOf traps, revocable. `test_jsengine_proxy_host.sh`.
  - **Typed arrays + ArrayBuffer** ‚Äî byte-backed (shared pool, little-endian pointer load/store): Int8/Uint8/Uint8Clamped/Int16/Uint16/Int32/Uint32/Float32/Float64. Construct from length / array / iterable / ArrayBuffer (shared view); indexed get/set with ToInt32+mask (clamp for Uint8Clamped); length/byteLength/byteOffset/buffer/BYTES_PER_ELEMENT; set()/subarray()/slice()/fill(); for-of/spread/Array.from; static of()/from(). Deferred: full Array method suite on views, DataView, BigInt64 arrays, endianness control. `test_jsengine_typedarray_host.sh`.
  - **structuredClone(value)** ‚Äî deep clone preserving shared refs + cycles; plain objects/arrays/Map/Set/Date/ArrayBuffer/typed arrays; functions/symbols throw DataCloneError. Deferred: accessor props, transfer list. `test_jsengine_structuredclone_host.sh`.
- **R8** (numeric literals + JSON reviver):
  - **Numeric literal forms** ‚Äî ES2021 numeric separators `_` in decimal/hex/binary/octal integers + fraction + exponent (`1_000_000`, `0xFF_FF`, `1e1_0`); binary `0b`/`0B` and octal `0o`/`0O` integer literals; BigInt `n` suffix (originally kept as a float64 Number ‚Äî **now a true arbitrary-precision BigInt as of R11**, see below). Fixed in `lib/web/js/lexer.ad` (lex_number) + `lib/web/js/value.ad` (parse_num_bytes). Previously the lexer split `1_000` into `1` + identifier `_000` (ReferenceError) and read `0b101` as `0` + identifier ‚Äî a hard abort of any modern minified/transpiled bundle using these forms. `test_jsengine_numliteral_host.sh`.
  - **JSON.parse(text, reviver)** ‚Äî the optional second argument is now honored: a bottom-up InternalizeJSONProperty walk calls `reviver.call(holder, key, value)` at every node (children first), deletes the property on an `undefined` result, handles arrays by index, threads the holder object as `this`, and invokes the root with key `""`. Previously the reviver was silently ignored (parsed data returned untransformed ‚Äî a correctness bug). Fixed in `lib/web/js/builtins/collections.ad` (json_parse_val + json_internalize/json_intern_props/json_reviver_call). `test_jsengine_numliteral_host.sh`. (JSON.stringify's `replacer` argument was subsequently landed in R10 ‚Äî see above.)
- **R9** (class private members + static blocks + global symbol registry):
  - **Class private `#fields` / `#methods` / accessors** ‚Äî the lexer now tokenizes `#name` as `TK_PRIVATE`, retaining the leading `#` so a private slot's key can never collide with a normal string property. Private instance fields (`#x = v`), private methods (`#m(){}`), private getters/setters (`get #x(){}`), and private static members (`static #n = v`) all parse and evaluate; `this.#x` / `Ctor.#n` access works through the ordinary member get/set path (stored under the `#`-prefixed key). The `#x in obj` **brand check** is a dedicated `ND_PRIVNAME` LHS handled by the `in` binop (never evaluated as a value; probes the private slot through the prototype chain). Crucially, private slots are HIDDEN: `Object.keys`, `for-in`, and `JSON.stringify` skip any key beginning with `#` (via a new `is_hidden_key` in `lib/web/js/value.ad`). Fixed across `lib/web/js/{lexer,parser,interp,value}.ad` + `builtins/collections.ad`. `test_jsengine_es2022priv_host.sh`.
  - **`Symbol.for(key)` / `Symbol.keyFor(sym)`** ‚Äî a process-wide symbol registry interns by string key so `Symbol.for("a") === Symbol.for("a")` (and stays distinct from a plain `Symbol("a")`); the registration key is stored back on the symbol as a hidden `@@forkey` own property for `Symbol.keyFor` reverse lookup (returns `undefined` for an unregistered symbol; throws `TypeError` on a non-symbol). Added in `lib/web/js/builtins/r7.ad` (symbol_for/symbol_key_for) + registered in `setup.ad`. `test_jsengine_es2022priv_host.sh`.
  - **`static { }` initializer blocks** ‚Äî the class-body parser now recognizes a bare `{ ... }` member after `static` (`ND_METHOD` with `n_c=3`); `eval_class` runs the block bodies interleaved with `static` fields in SOURCE ORDER, with `this` = the class object and the class's own name bound in a class-scope env (so `static { C.x = ... }`, `static y = C.z`, and `this.#priv` inside a block all resolve). Added in `lib/web/js/parser.ad` (parse_class) + `interp.ad` (eval_class static pass). `test_jsengine_es2022priv_host.sh`.
  - **Bonus fix (symbol-key leak)** ‚Äî the same `is_hidden_key` filter also stops symbol-keyed properties (`@@sym:` keys) from leaking into `Object.keys` / `JSON.stringify`, a pre-existing correctness bug (`o[sym]=v` previously showed `@@sym:NNN` in enumeration output).
- **R10** (`for await...of` + JSON.stringify replacer):
  - **`for await (x of asyncIterable)`** async iteration ‚Äî the parser recognizes a contextual `await` after `for` (inside an async body) and marks `ND_FOROF` (`n_d=1`); `exec_forof` dispatches to a new `exec_forawait` (`lib/web/js/interp.ad`). It first tries the source's `@@asyncIterator` (a fresh global `g_sym_async` symbol, also installed on the generator prototype so an `async function*` is its own async iterator): it calls `next()`, **awaits** the result via the engine's existing deterministic `do_await` (microtask-drain) to get `{value, done}`, and binds/executes the loop body per element (honoring `break`/`continue`/labels through the shared `forof_bind`/`forof_after`). If there is no `@@asyncIterator`, it falls back to the **async-from-sync wrapper**: it materializes the sync iterable via `collect_iterable_into` and **awaits each produced value** ‚Äî so `for await (x of [Promise.resolve(1), 2, ...])` yields the resolved values in order. Because `await` drains the deterministic queue, the whole loop completes synchronously (no call-stack suspension), matching the engine's eager async/generator model. `test_jsengine_asynciter_host.sh`.
  - **`JSON.stringify(value, replacer[, space])`** ‚Äî the 2nd argument is now honored. A **function replacer** `fn(key, value)` is called (with the holder as `this`) per property, its result substituted; an **array replacer** is an allowlist of object keys to include (applied at every object level, not to array indices). `value.toJSON(key)` is also honored on this path. Because a replacer function does its own string-building (which cannot run while the shared build cursor is held by `json_emit` ‚Äî the same two-phase hazard as `String.replace`), the value tree is **pre-transformed** into a fresh tree (`json_transform`/`json_rep_value`/`json_key_allowed`, root holder `{"": value}`) BEFORE the emitter runs; the no-replacer path stays on the fast direct emitter (byte-identical to before). Added in `lib/web/js/builtins/collections.ad` (`json_stringify_full` + helpers). `test_jsengine_asynciter_host.sh`.
- **R11** (true arbitrary-precision BigInt ‚Äî closes the largest ES gap):
  - **BigInt value type** ‚Äî a new `TAG_BIGINT` value tag (`lib/web/js/consts.ad`) whose `v_ref` indexes a new immutable bignum record: a **sign + base-2^32 little-endian limb magnitude** (`lib/web/js/bigint.ad`, pools `bi_sign`/`bi_off`/`bi_len` over a shared `bi_limbs` uint32 bump pool). Records are immutable, so a sign flip shares limbs with no copy. `123n` / `0xffn` / `0b1010n` / `0o17n` literals now LEX to a distinct `TK_BIGINT` token carrying the source digits (lexer keeps the text sans `n`; the regex/divide heuristic treats a BigInt literal as a value so `7n / 2n` is division, not a regex) and PARSE to `ND_BIGINT`, evaluated by exact `bi_parse` ‚Äî no more float64 precision loss past 2^53.
  - **Arithmetic** ‚Äî exact `+ - * / % **` on two BigInts: schoolbook magnitude add/sub/mul, **bit-by-bit long division** (√∑ **truncates toward zero**, `%` takes the dividend's sign), and exponentiation-by-squaring. `++`/`--` and compound `+= -= *= ‚Ä¶` stay BigInt. Mixing a BigInt with a Number in an arithmetic/bitwise op throws **TypeError** (per spec); unary `+` on a BigInt throws; `/ 0n` and a negative `**` exponent throw **RangeError**.
  - **Comparisons** ‚Äî `< > <= >=` and `== ===` including exact **BigInt-vs-Number numeric compare** (`2n < 3`, `5n == 5` true, `5n === 5` false ‚Äî `===` is false across types); loose `==` also handles BigInt-vs-String/Boolean by mathematical value.
  - **Bitwise** ‚Äî `& | ^` implemented with **infinite-width two's-complement** semantics (correct for negative operands via a to/from two's-complement conversion); `<<` is a magnitude shift keeping sign; `>>` is an **arithmetic (floor) shift** (`-8n >> 1n === -4n`, `-7n >> 1n === -4n`); `~x === -(x+1)`. `>>>` throws TypeError (BigInts have no unsigned shift).
  - **Conversions / methods** ‚Äî `typeof 1n === "bigint"`; `String(1n)` / `` `${1n}` `` / `1n.toString(radix)` (2‚Äì36); truthiness (`0n` falsy); `BigInt(number|string|boolean|bigint)` (non-integer Number ‚Üí RangeError); `BigInt.asIntN(bits,x)` / `BigInt.asUintN(bits,x)`. `JSON.stringify` on a BigInt (or an object containing one) throws **TypeError**, per spec. Console/display prints the digits with no trailing `n` (String() semantics). Touched: `lib/web/js/{bigint,consts,state,lexer,parser,interp,setup}.ad` + `builtins/{collections,strings}.ad`. `test_jsengine_bigint_host.sh` (40 asserts vs known results: `2n**100n`, `50!`, div/mod signs, two's-complement bitwise, mixing/JSON TypeErrors).
  - Coverage is honest and broad: bitwise AND/OR/XOR/shifts (incl. negatives), `asIntN`/`asUintN`, and all six arithmetic ops are DONE. NOT done: `BigInt64Array`/`BigUint64Array` typed arrays, `DataView` get/setBigInt64, and a `Number(bigint)` that round-trips huge magnitudes exactly (uses a float64 approximation for out-of-int64 comparison fallbacks only).
  - Next JS gap: async iterator `return()`/`throw()` propagation and `Symbol.asyncIterator` on user objects returning promises from `next()` (partially exercised; async generators covered); then `BigInt64Array`/`DataView` BigInt accessors.
- **R12** (destructuring ASSIGNMENT + `window`-shaped globals):
  - **Destructuring as an ASSIGNMENT statement** ‚Äî `[a,b]=[b,a]` (swap), `[x,,y]=arr` (elision/skip), `[a,...rest]=arr` (rest), `({p,q}=obj)`, `({name:nm}=obj)` (rename), object rest `({a,...others}=obj)`, element/property **defaults** (`[m=5]`, `{v=5}`), nested patterns (`[p,[q,r]]=‚Ä¶`, `({user:{id,tags:[t0,t1]}}=‚Ä¶)`), and member/index leaf targets (`[o.x,a[0]]=‚Ä¶`) all work. Previously this did NOTHING: `assign_to` handled only ident/member/index LHS, so an array/object **literal** LHS was silently ignored ‚Äî only the `let/const/var` DECLARATION path (`bind_pattern`) understood patterns. Root cause + fix: added `assign_array_pattern`/`assign_object_pattern` in `lib/web/js/interp.ad`, dispatched from `assign_to` on an `ND_ARRAY`/`ND_OBJECT` target; they walk the literal's element list and assign each leaf via `assign_to` (recursing for nested patterns). Because `eval_assign` already fully evaluates the RHS before calling `assign_to`, the swap is correct. Two supporting parser fixes (`lib/web/js/parser.ad`): array-literal **elisions are now preserved** as a new `ND_HOLE` node (`parse_array_lit`) so `[x,,y]=arr` maps `y‚Üêarr[2]` (a hole also now reads as `undefined` with correct `.length` in a value context, via `eval_array_lit`); and object shorthand **CoverInitializedName** `{x = default}` parses (wrapping the shorthand in an `ND_ASSIGN`) so `({v=5}=obj)` applies its default. Declaration destructuring is unchanged and re-asserted green. `test_jsengine_destructure_host.sh`.
  - **`window`-shaped globals** ‚Äî the existing `window`/`globalThis`/`self` alias now also carries the browser viewport/URL data properties pages read ungated: `innerWidth`/`innerHeight`/`outerWidth`/`outerHeight` (default 1024√ó768), `scrollX`/`scrollY`/`pageXOffset`/`pageYOffset` (0), `devicePixelRatio`/`screenX`/`screenY`, a minimal `location` object (`href`/`protocol`/`host`/`hostname`/`port`/`pathname`/`search`/`hash`/`origin`), and a `screen` object. Defined on the global scope (like `navigator`) so both `window.innerWidth` and bare `innerWidth` resolve. Added in `lib/web/js/setup.ad`. DEFERRED (crosses into the DOM layer this JS track does not own): `window.addEventListener('load'|'DOMContentLoaded', ‚Ä¶)` / `window.onload` firing ‚Äî the load lifecycle is dispatched by `lib/web/dom/canvas.ad` `_fire_document` against `document`, so wiring `window` listeners into it belongs there. `test_jsengine_destructure_host.sh`.
