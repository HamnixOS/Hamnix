# Native Browser (hambrowse) — W3C / ECMAScript Conformance Roadmap

From a 14-agent parallel audit (2026-07-16). Dual-target, host-iterated (~40 gates). ~30-50% conformant/area; full conformance needs real spec ALGORITHMS per subsystem — enabled by the modular lib/web/ tree refactor.

## ECMAScript rounds landed (lib/web/js/*, host gates in scripts/test_jsengine_*_host.sh)

- R1-6: templates/tagged templates, classes + fields + accessors, Symbol + well-known symbols, change-array-by-copy, regex named groups, generators (`function*`/`yield`/`yield*`, bounded-eager model) + the iteration protocol, Map/Set, optional chaining/nullish, for-of, Object/Reflect-ish statics.
- **R7** (this round):
  - **Generator methods** — `*name(){}` in object literals and class bodies (incl. `static *m(){}`). `test_jsengine_generators_host.sh`.
  - **Reflect** — get/set/has/ownKeys/getPrototypeOf/setPrototypeOf/defineProperty/deleteProperty/apply/construct (thin wrappers over core ops). Also fixed the `in` operator to walk the prototype chain. `test_jsengine_reflect_host.sh`.
  - **Proxy** — `new Proxy(target, handler)` with the get/set/has/deleteProperty/ownKeys traps routed through member-get/set, `in`, `delete`, Object.keys/Reflect.ownKeys; absent trap forwards to target. Deferred: apply/construct/defineProperty/getOwnPropertyDescriptor/get(set)PrototypeOf traps, revocable. `test_jsengine_proxy_host.sh`.
  - **Typed arrays + ArrayBuffer** — byte-backed (shared pool, little-endian pointer load/store): Int8/Uint8/Uint8Clamped/Int16/Uint16/Int32/Uint32/Float32/Float64. Construct from length / array / iterable / ArrayBuffer (shared view); indexed get/set with ToInt32+mask (clamp for Uint8Clamped); length/byteLength/byteOffset/buffer/BYTES_PER_ELEMENT; set()/subarray()/slice()/fill(); for-of/spread/Array.from; static of()/from(). Deferred: full Array method suite on views, DataView, BigInt64 arrays, endianness control. `test_jsengine_typedarray_host.sh`.
  - **structuredClone(value)** — deep clone preserving shared refs + cycles; plain objects/arrays/Map/Set/Date/ArrayBuffer/typed arrays; functions/symbols throw DataCloneError. Deferred: accessor props, transfer list. `test_jsengine_structuredclone_host.sh`.
- **R8** (numeric literals + JSON reviver):
  - **Numeric literal forms** — ES2021 numeric separators `_` in decimal/hex/binary/octal integers + fraction + exponent (`1_000_000`, `0xFF_FF`, `1e1_0`); binary `0b`/`0B` and octal `0o`/`0O` integer literals; BigInt `n` suffix consumed (value kept as a float64 Number — enough to keep scripts running, NOT true arbitrary-precision BigInt). Fixed in `lib/web/js/lexer.ad` (lex_number) + `lib/web/js/value.ad` (parse_num_bytes). Previously the lexer split `1_000` into `1` + identifier `_000` (ReferenceError) and read `0b101` as `0` + identifier — a hard abort of any modern minified/transpiled bundle using these forms. `test_jsengine_numliteral_host.sh`.
  - **JSON.parse(text, reviver)** — the optional second argument is now honored: a bottom-up InternalizeJSONProperty walk calls `reviver.call(holder, key, value)` at every node (children first), deletes the property on an `undefined` result, handles arrays by index, threads the holder object as `this`, and invokes the root with key `""`. Previously the reviver was silently ignored (parsed data returned untransformed — a correctness bug). Fixed in `lib/web/js/builtins/collections.ad` (json_parse_val + json_internalize/json_intern_props/json_reviver_call). `test_jsengine_numliteral_host.sh`. NOTE STILL MISSING: JSON.stringify's `replacer` (function/array allowlist) argument is still ignored — next JS round.
- **R9** (class private members + static blocks + global symbol registry):
  - **Class private `#fields` / `#methods` / accessors** — the lexer now tokenizes `#name` as `TK_PRIVATE`, retaining the leading `#` so a private slot's key can never collide with a normal string property. Private instance fields (`#x = v`), private methods (`#m(){}`), private getters/setters (`get #x(){}`), and private static members (`static #n = v`) all parse and evaluate; `this.#x` / `Ctor.#n` access works through the ordinary member get/set path (stored under the `#`-prefixed key). The `#x in obj` **brand check** is a dedicated `ND_PRIVNAME` LHS handled by the `in` binop (never evaluated as a value; probes the private slot through the prototype chain). Crucially, private slots are HIDDEN: `Object.keys`, `for-in`, and `JSON.stringify` skip any key beginning with `#` (via a new `is_hidden_key` in `lib/web/js/value.ad`). Fixed across `lib/web/js/{lexer,parser,interp,value}.ad` + `builtins/collections.ad`. `test_jsengine_es2022priv_host.sh`.
  - **`Symbol.for(key)` / `Symbol.keyFor(sym)`** — a process-wide symbol registry interns by string key so `Symbol.for("a") === Symbol.for("a")` (and stays distinct from a plain `Symbol("a")`); the registration key is stored back on the symbol as a hidden `@@forkey` own property for `Symbol.keyFor` reverse lookup (returns `undefined` for an unregistered symbol; throws `TypeError` on a non-symbol). Added in `lib/web/js/builtins/r7.ad` (symbol_for/symbol_key_for) + registered in `setup.ad`. `test_jsengine_es2022priv_host.sh`.
  - **`static { }` initializer blocks** — the class-body parser now recognizes a bare `{ ... }` member after `static` (`ND_METHOD` with `n_c=3`); `eval_class` runs the block bodies interleaved with `static` fields in SOURCE ORDER, with `this` = the class object and the class's own name bound in a class-scope env (so `static { C.x = ... }`, `static y = C.z`, and `this.#priv` inside a block all resolve). Added in `lib/web/js/parser.ad` (parse_class) + `interp.ad` (eval_class static pass). `test_jsengine_es2022priv_host.sh`.
  - **Bonus fix (symbol-key leak)** — the same `is_hidden_key` filter also stops symbol-keyed properties (`@@sym:` keys) from leaking into `Object.keys` / `JSON.stringify`, a pre-existing correctness bug (`o[sym]=v` previously showed `@@sym:NNN` in enumeration output).
  - STILL genuinely missing (next JS rounds): `for await...of` async iteration, true arbitrary-precision BigInt, `JSON.stringify` `replacer` argument.
