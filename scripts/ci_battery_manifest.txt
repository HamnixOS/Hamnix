# scripts/ci_battery_manifest.txt — the bare-metal regression battery.
#
# One gate invocation per line, in shortest-first order. The CI job
# 'hamnix-bare-metal' round-robin-shards this list across a matrix of
# parallel runners (index % NSHARDS), so all gates keep running on every
# push but each shard finishes well under its timeout — 56 pure-TCG QEMU
# boots serially blew past the 40-min budget (~75 min needed) and the
# whole battery went permanently red, providing ZERO signal. Add a gate:
# drop a line here (order = fast→slow keeps shards balanced under round-
# robin). Wrapped gates (ci_run_gate.sh) emit the three-valued verdict
# (INCONCLUSIVE→warning on a TCG-starved runner); direct ones are hard
# PASS/FAIL. Blank lines and # comments are ignored.

# test_ahci_blk, test_ahci_recovery and test_blk_sched used to deterministically
# WEDGE the guest the first time they drove a cold blk_read/blk_write (generic
# ahci_read/write_sectors -> _ncq_block_rw_sync, the SLEEPING NCQ FPDMA path)
# against the active AHCI port AFTER sched_init. Root cause: the driver set
# GHC.AE but NEVER GHC.IE — the GLOBAL AHCI interrupt-enable gate (spec §3.1.2).

# survived because the blkpath selftest busy-polls _ncq_irq_complete_port to
# drain, needing neither IRQ nor timeout; cold submitted and slept forever.
