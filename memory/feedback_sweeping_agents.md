User: *"Don't shy away from large topics that need a single agent to do sweeping changes. Let's push this into top tier OS/kernel."*

The fork() keystone, COW, higher-half kernel, namespace reworks — all solved by one solo agent with full authority. Narrow fenced parallel dispatches stalled on them.

**Apply:** when a topic is genuinely large and cross-cutting, dispatch ONE agent with sweeping whole-worktree authority — including normally-fenced files — rather than splitting into narrow tasks that lack scope to finish.

Sweeping ≠ sloppy. Still respect worktree isolation, orchestrator-merges, no-regression bar. Still let the agent run wild ([[feedback-let-agents-run-wild]]).
